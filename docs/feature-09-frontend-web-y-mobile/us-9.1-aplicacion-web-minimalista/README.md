# User Story 9.1 - Aplicación web minimalista

**Feature:** 9 - Frontend web y mobile

**Tipo:** Frontend

## Historia

**Como** usuario

**Quiero** usar una aplicación web simple que consuma la API común

**Para** gestionar mi portfolio desde el navegador con una experiencia consistente.

## Criterios de aceptación

1. Dado un usuario, cuando accede a la web, entonces puede usar las funcionalidades principales desde navegador.
2. Dado la interfaz, cuando se navega, entonces mantiene un diseño simple y consistente.
3. Dado errores de API, cuando ocurren, entonces la web muestra mensajes claros.
4. Dado una sesión activa, cuando el usuario navega entre pantallas, entonces mantiene el estado.
5. Dado la web, cuando realiza login, portfolio, historial o watchlist, entonces consume la API común.
6. Dado una respuesta exitosa de API, cuando llega al frontend, entonces se renderiza correctamente.
7. Dado una respuesta de error, cuando llega al frontend, entonces se informa al usuario.
8. Dado cambios de datos, cuando se actualizan desde web, entonces persisten en backend.
