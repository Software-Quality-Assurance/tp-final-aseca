# User Story 9.4 - Pantallas principales del producto

**Feature:** 9 - Frontend web y mobile

**Tipo:** Frontend

## Historia

**Como** usuario

**Quiero** utilizar pantallas claras para portfolio, valor actual, historial y watchlist

**Para** gestionar y analizar mis inversiones desde una interfaz centralizada.

## Criterios de aceptación

1. Dado un usuario autenticado, cuando abre Portfolio Page, entonces ve sus posiciones.
2. Dado el botón de agregar, editar o eliminar, cuando lo selecciona, entonces se abre el modal correspondiente.
3. Dado accesos a valor actual, historial y watchlist, cuando los selecciona, entonces navega correctamente.
4. Dado un portfolio con posiciones, cuando se abre Valor Actual Page, entonces muestra valor total, P&L por acción y timestamp de precios.
5. Dado portfolio vacío, cuando se abre Valor Actual Page, entonces muestra estado vacío sin error.
6. Dado operaciones existentes, cuando se abre Historial Page, entonces se muestran en un cuadro claro.
7. Dado cada operación, cuando se muestra, entonces incluye tipo, compañía, cantidad, precio y fecha.
8. Dado historial vacío, cuando se abre, entonces muestra estado vacío.
9. Dado navegación de regreso, cuando se selecciona Portfolio Page, entonces vuelve correctamente.
10. Dado una watchlist, cuando se abre Watchlist Page, entonces muestra empresas agregadas.
11. Dado el botón agregar, cuando se selecciona, entonces permite agregar empresa desde modal.
12. Dado una empresa existente en watchlist, cuando se elimina, entonces desaparece de la lista.
13. Dado comparación disponible, cuando se solicita, entonces muestra métricas financieras comparables.
