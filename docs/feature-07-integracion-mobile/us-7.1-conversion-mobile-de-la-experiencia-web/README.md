# User Story 7.1 - Conversión mobile de la experiencia web

**Feature:** 7 - Integración mobile

**Tipo:** Frontend Mobile

## Historia

**Como** usuario

**Quiero** contar en mobile con los mismos flujos principales que uso en la web

**Para** gestionar mi portfolio desde el celular con una experiencia consistente y conectada a la misma API.

## Criterios de aceptación

1. Dado un usuario, cuando accede a la app mobile, entonces puede usar las funcionalidades principales disponibles en web.
2. Dado la interfaz mobile, cuando se navega, entonces mantiene un diseño simple, consistente y adaptado al tamaño de pantalla.
3. Dado errores de API, cuando ocurren, entonces mobile muestra mensajes claros.
4. Dado una sesión activa, cuando el usuario navega entre pantallas mobile, entonces mantiene el estado.
5. Dado mobile, cuando realiza login, portfolio, historial o watchlist, entonces consume la API común.
6. Dado una respuesta exitosa de API, cuando llega a mobile, entonces se renderiza correctamente.
7. Dado una respuesta de error, cuando llega a mobile, entonces se informa al usuario.
8. Dado cambios de datos desde mobile, cuando se actualizan, entonces persisten en backend y son visibles desde web.
