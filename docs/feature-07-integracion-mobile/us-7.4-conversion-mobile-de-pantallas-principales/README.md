# User Story 7.4 - Conversión mobile de pantallas principales

**Feature:** 7 - Integración mobile

**Tipo:** Frontend Mobile

## Historia

**Como** usuario

**Quiero** usar en mobile las pantallas de portfolio, valor actual, historial y watchlist equivalentes a la web

**Para** gestionar y analizar mis inversiones desde una interfaz adaptada al celular.

## Criterios de aceptación

1. Dado un usuario autenticado en mobile, cuando abre Portfolio Page, entonces ve sus posiciones.
2. Dado el botón de agregar, editar o eliminar en mobile, cuando lo selecciona, entonces se abre el modal o pantalla correspondiente.
3. Dado accesos a valor actual, historial y watchlist en mobile, cuando los selecciona, entonces navega correctamente.
4. Dado un portfolio con posiciones, cuando se abre Valor Actual Page en mobile, entonces muestra valor total, P&L por acción y timestamp de precios.
5. Dado portfolio vacío, cuando se abre Valor Actual Page en mobile, entonces muestra estado vacío sin error.
6. Dado operaciones existentes, cuando se abre Historial Page en mobile, entonces se muestran en un formato legible.
7. Dado cada operación, cuando se muestra en mobile, entonces incluye tipo, compañía, cantidad, precio y fecha.
8. Dado historial vacío, cuando se abre en mobile, entonces muestra estado vacío.
9. Dado navegación de regreso, cuando se selecciona Portfolio Page en mobile, entonces vuelve correctamente.
10. Dado una watchlist, cuando se abre Watchlist Page en mobile, entonces muestra empresas agregadas.
11. Dado el botón agregar en mobile, cuando se selecciona, entonces permite agregar empresa desde modal o pantalla adaptada.
12. Dado una empresa existente en watchlist, cuando se elimina desde mobile, entonces desaparece de la lista.
13. Dado comparación disponible, cuando se solicita desde mobile, entonces muestra métricas financieras comparables en un formato legible.
